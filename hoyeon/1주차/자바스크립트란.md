# 1장 자바스크립트

## 자바스크립트

‘웹페이지에 생동감을 불어넣기 위해’ 만들어진 프로그래밍 언어

**<장점>**

- 브라우저 뿐만 아니라 서버에서도 돌릴 수 있는 언어
- HTML/CSS와 완전히 통합할 수 있음
- 간단한 일은 간단하게 처리할 수 있게 해줌
- 모든 주요 브라우저에서 지원하고, 기본 언어로 사용됨

<자바스크립트 엔진>

- 자바스크립트 언어를 읽고 해석할 수 있는 도구 (프로그램)
  브라우저엔 '자바스크립트 가상 머신’이라 불리는 엔진이 내장

V8 – Chrome, Opera
SpiderMonkey – Firefox
ChakraCore - Microsoft Edge
SquirrelFish - Safari

- 엔진 동작 방식
  - 엔진이 스크립트를 해석(파싱)
  - 해석한 스크립트를 기계어로 전환(컴파일)
  - 기계어로 전환된 코드가 실행 (기계어로 전환되었기 때문에 실행 속도가 빠름.)
  - 엔진은 프로세스 각 단계마다 최적화를 진행
    심지어 컴파일이 끝나고 실행 중인 코드를 감시하면서, 이 코드로 흘러가는 데이터를 분석하고, 분석 결과를 토대로 기계어로 전환된 코드를 다시 최적화함.
    이 과정을 거치면 스크립트 실행 속도는 더욱 더 빨라짐.

### 브라우저에서 자바스크립트로 할 수 있는 일

**웹페이지 조작**: 새로운 HTML 추가, 기존 HTML 및 스타일 수정
**사용자 행동에 반응**: 마우스 클릭, 포인터 움직임, 키보드 입력 등 반응
**서버와의 상호작용**: AJAX나 COMET 같은 기술로 원격 서버에 요청 보내기, 파일 다운로드/업로드
**데이터 관리**: 쿠키 가져오기/설정, 로컬 스토리지에 데이터 저장
**사용자 인터랙션**: 질문을 건네거나 메시지 보여주기

### 브라우저에서 할 수 없는 일

**파일 시스템 접근 제한**: 웹페이지 스크립트는 임의의 파일 읽기, 쓰기, 복사, 실행이 제한됨
**하드웨어 접근 제한**: 카메라나 마이크 같은 장치에 접근하려면 사용자 허가 필요
**탭 간 데이터 공유 제한**: 동일 도메인, 프로토콜, 포트를 사용하지 않는 한 다른 창 또는 탭과의 데이터 교환 제한
**동일 출처 정책**: 다른 도메인에서 데이터를 받아오는 데 제약이 있으며, 원격 서버의 명확한 승인(HTTP 헤더 등)이 필요

---

자바스크립트는 모든 프로젝트 요구사항을 충족시키기 어렵기 때문에, 자바스크립트로 트랜스파일(변환)할 수 있는 다양한 언어들이 등장

CoffeeScript: 자바스크립트의 짧고 명료한 문법을 제공. Ruby 개발자들이 선호
TypeScript: 자료형을 명시하여 복잡한 시스템을 지원. Microsoft 개발
Flow: 자료형 강제를 통해 개발 단순화. Facebook 개발
Dart: 모바일 앱 개발 등 브라우저 외 환경을 위한 언어. Google 개발
